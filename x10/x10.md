# 0x10. 다이나믹 프로그래밍

## 알고리즘 설명

### 다이나믹 프로그래밍 (Dynamic Programming)
- 여러 개의 하위 문제를 먼저 푼 후 그 결과를 쌓아올려 주어진 문제를 해결하는 알고리즘
- **점화식을 찾은 후, 점화식의 항을 밑에서부터 하나씩 구해서 답을 알아내는 알고리즘**
- ex. 피보나치 수열
    - 재귀로 해결하면 중복되는 값을 연산해서 속도가 느림
    - DP로 풀면 배열을 만들어두고 채워나가면 됨 (N + 1칸을 채우면 답을 도출할 수 있어 O(N))

### DP를 푸는 과정
- 테이블 정의하기
- 점화식 찾기 (제일 중요함)
- 초기값 정하기
+ DP 문제라는 것을 알아차리는 것부터 어려움

## 연습 문제들

### BOJ 1463번: 1로 만들기
- BFS로 풀 수 있음 (집가서 풀어보기)

- 테이블 정의하기
    - D[i] = i를 1로 만들기 위해 필요한 연산 사용 횟수의 최솟값

- 점화식 찾기
    - D[12] = ?
    - D[1] ~ D[11]까지의 값을 알고 있다는 가정
    - D[12]에서 할 수 있는 연산 3가지
        - 3으로 나누거나 (D[12] = D[4] + 1)
        - 2로 나누거나 (D[12] = D[6] + 1)
        - 1을 빼거나 (D[12] = D[11] + 1)
        - D[12] = (min(D[4], D[6], D[11]) + 1)
    
- 점화식 찾기
    - D[k]
    - 3으로 나누어지면 3으로 나누거나 (D[k] = D[k/3] + 1)
    - 2로 나누어지면 2로 나누거나 (D[k] = D[k/2] + 1)
    - 1을 빼거나 (D[k] = D[k-1] + 1)

- 초기값 정의하기
    - D[1] = 0;

- 구현
```cpp
#include <bits/stdc++.h>
using namespace std;

int N;
int arr[1000005];

int main(void) {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    cin >> N;
    for (int i = 2; i <= N; i++) {
        arr[i] = arr[i-1] + 1;
        if (i % 2 == 0) arr[i] = min(arr[i], arr[i/2] + 1);
        if (i % 3 == 0) arr[i] = min(arr[i], arr[i/3] + 1);
    }
    cout << arr[N] << '\n';
    return 0;
}
```

### BOJ 9095번: 1, 2, 3 더하기
- 테이블 정의하기
    - D[i] = i를 1, 2, 3합으로 나타내는 방법의 수

- 점화식 찾기
    - D[4] = ?
    - 1+1+1+1, 3+1, 2+1+1, 1+2+1 (3을 1, 2, 3합으로 나타내는 방법의 수) + 1 / D[3]
    - 1+1+2, 2+2 (2를 1, 2, 3합으로 나타내는 방법의 수) + 2 / D[2]
    - 1+3 (1을 1, 2, 3합으로 나타내는 방법의 수) + 3 / D[1]
    - D[4] = D[3] + D[2] + D[1]
    - D[i] = D[i-1] + D[i-2] + D[i-3]

- 초기값 정의하기
    - D[1] = 1, D[2] = 2, D[3] = 4
    - D[i] = D[i-1] + D[i-2] + D[i-3]이므로 초기값이 최소 3개는 주어져야함

- 구현
```cpp
#include <bits/stdc++.h>
using namespace std;

int T; // <= 10
int N;
int D[11];

int main(void) {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    D[1] = 1;
    D[2] = 2;
    D[3] = 4;
    for (int i = 4; i <= 11; i++) {
        D[i] = D[i-1] + D[i-2] + D[i-3];
    }

    cin >> T;
    for (int i = 0; i < T; i++) {
        cin >> N;
        cout << D[N] << '\n';
    }

    return 0;
}

// N이 주어질 때마다 새로 구하는 것보다 어차피 미리 구하고 꺼내서 쓸 수 있는 상황이면 이렇게 구현
```

### BOJ 2579번: 계단 오르기
- 테이블 정의하기
    - D[i] = i를 해당 최대 점수

- 점화식 찾기
    - D[6] = ?
    - D[6] = D[]

+ 내가 현재 생각했던 방법으로 점화식을 찾아내는 것은 제약 조건때문에 안됨

- 테이블 정의하기
    - D[i][j] = 현재까지 j개의 계단을 연속해서 밟고, i번째 계단까지 올라갔을 때
    - 점수 합의 최댓값, 단 i번째 계단은 반드시 밟아야함

- 점화식 찾기
    - i일 경우 j는 1, 2 중 하나 (3은 안됨)
    - D[i][1] / D[i][2]
    - D[k][1]을 먼저
        - 현재까지 1개의 계단을 연속해서 밟고 k번째 계단까지 올라갔을 때 점수의 최대 합
        - k-1번째 계단을 밟지 않았음을 의미함

    - D[k][1] = max(D[k-2][1], D[k-2][2]) + S[k];

    - D[k][2]
        - 현재까지 2개의 계단을 연속해서 밟고 k번째 계단까지 올라갔을 때 점수의 최대 합
        - k-1번째 계단을 밟았음을 의미함

    - D[k][2] = D[k-1][1] + S[k];

    - 도착: max(D[N][1], D[N][2])

- 초기값 정의하기
    - D[1][0] / D[1][1] / D[2][0] / D[2][1]
    - D[1][1] = S[1]
    - D[1][2] = 0
    - D[2][1] = S[2]
    - D[2][2] = S[1] + S[2]

- 구현
```cpp
#include <bits/stdc++.h>
using namespace std;

int T; // 도착 계단 위치 / 1 <= T <= 300
int S[305];
int D[305][3];

int main(void) {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    cin >> T;
    for (int i = 1; i <= T; i++) {
        cin >> S[i];
    }

    if (T == 1) {
        cout << S[1] << '\n';
        return 0;
    }
    
    D[1][1] = S[1];
    D[1][2] = 0;
    D[2][1] = S[2];
    D[2][2] = S[1] + S[2];

    for (int i = 3; i <= T; i++) {
        D[i][1] = max(D[i-2][1], D[i-2][2]) + S[i];
        D[i][2] = D[i-1][1] + S[i];
    }

    cout << max(D[T][1], D[T][2]) << '\n';

    return 0;
}

// 일단 약간의 고찰: 초기값 정의하기에서 점화식 자체도 중요한데, 일부 손으로 구해야하는 것도 있다는 점을 인지
```
