## 시간, 공간 복잡도

### 시간 제한 1초
- 컴퓨터는 1초에 대략 3-5억개의 연산을 처리할 수 있음 (단순한 연산-복잡한 연산)

### 정렬되어있다는 가정하에 절반씩 줄여가면 logN
- N * (1/2)^k = 1
- N = 2^k
- k = logN

### 시간 복잡도(Time Complexity)
- 입력의 크기(N)와 문제를 해결하는 걸리는 시간(k)의 상관관계

### 빅오표기법(Big-O Notation)
- 주어진 식을 값이 가장 큰 대표항만 남겨서 나타내는 방법
    - 그 이유는 입력의 크기가 커지면 뒤에는 의미없는 수치이기 때문에(엄밀하진 않음)
- **절대적인 시간을 측정하는 개념이 아님 / 입력의 크기(N) 증가에 따른 시간의 변화 추리를 보는 것**
- O(N) = 5N + 3, 2N + 10
- O(N^2) = N^2 + 2N + 4
- O(NlgN) = NlnN + 30N
- O(1) = 5, 16, 36 (상수 시간)
    - 오해가 있을 수도 있는데, 데이터가 적을 땐, O(N) 이런게 더 빠를수도 있음
    - 정확한 의미는 입력의 크기에 상관 없이 항상 같은 시간이 걸리다는 의미가 중요함

### 빅오표기법에 따른 시간 복잡도 크기 순서
- O(1) < O(lgN) < O(N) < O(NlgN) < O(N^2) < O(2^N) < O(N!)
- O(1):     상수 시간
- O(lgN):   로그 시간
- O(N):     선형 시간
- 나머지:    다항 시간

### 항상 생각할 점
- 문제를 보고, 풀이를 떠올린 후에 무턱대고 바로 짜는게 아니라
- 내 풀이가 이 문제를 제한 시간 내로 통과할 수 있는지, 시간 복잡도가 올바른지 꼭 생각해봐야함

### 시간 복잡도 가늠하기
- $N \le 20$: $O(2^N)$ 또는 $O(N!)$ (완전 탐색)
- $N \le 100$: $O(N^4)$
- $N \le 500$: $O(N^3)$
- $N \le 5,000$: $O(N^2)$
- $N \le 1,000,000$: $O(N \log N)$
- $N \le 10,000,00$: $O(N)
- $N$이 그 이상: $O(\log N)$ 또는 $O(1)$

### 공간 복잡도
- 입력의 크기와 문제를 해결하는데 필요한 공간의 상관관계
- 512MB = 1.2억개의 int
- 배열이 필요없으면 O(1) / O(N): 1차원 / O(N^2): 2차원
- int배열 5억개 필요 -> 512MB (X)
