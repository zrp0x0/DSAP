## STL과 함수 인자

### STL - vector
- 주의할점: 
    - vector는 복사됨
```cpp
bool cmp1(vector<int> v1, vector<int> v2, int idx) {
    return v1[idx] > v2[idx];
} 
- 이거 시간 복잡도는 무엇일까?
    - O(N)
    - v1, v2를 인자로 실어서 보낼 때 원본으로부터 복사본을 만드는 비용을 생각하지 못하는 경우가 있음
    - 복사 비용도 시간 복잡도에 넣어서 생각해야함

- 즉, 하고 싶은 말은 복사되지 않게 조심해라!!
    - 즉, 주솟값을 넘겨라
```cpp

```
bool cmp1(vector<int>& v1, vector<int>& v2, int idx) {
    return v1[idx] > v2[idx];
}

## 표준 입출력

### cin / cout
- scanf / printf는 C++ String을 처리할 수 없음

### scanf / cin의 주의점
- 공백이 포함된 문자열은 입력받을 수 없음
```cpp
char a1[10];
scanf("%[^\n]", a1);

string s;
getline(cin, s);
```

### ios::sync_with_stdio(0) / cin.tie(0)
- cin/cout은 입출력으로 인한 시간 초과를 막기 위해서 위 두 명령어를 반드시 실행시켜야함
- 이걸 안하면 입출력 양이 많을 때 시간초과가 날 수 있음
- 간단히 이야기하자면, C++ Stream만 사용할거면 C Stream이랑 동기화할 
    - 0 -> false가 더 맞는 표현

- cin.tie(0)은 버퍼랑 관련있는데
    - 기본적으로 cin은 cout이 실행되면 버퍼를 비우는 동작을 수행함
    - 근데 어차피 채점에서는 console에 출력되는 것만 보기 때문에 상관없음
    - 그니깐 무슨 말이냐면, 입력-출력으로 나오는게 보통 생각인데, 버퍼를 비우지 않으면 입력-입력-출력-출력 이런식으로 나올 수도 있다는 거임
    - 그래서 cin 다음에 cout이 오면 버퍼를 강제로 비움
    - 그걸 그냥 하지 말도록 하는게 cin.tie(nullptr)
